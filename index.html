HTML

  • Element refers to the opening and closing tag.
  • The tag refers to the text inside the angle brackets
  • Not all elements have a closing tag and are known as self-closing tags
  • A closing tag begins with a /
  • The !DOCTYPE html declares to the browser the document type is html.
  • Best practice to use semantic elements when possible, especially when visible to users.
  • The head element contains metadata that web browsers use to bring the webpage to the user.
  • The meta element defines the metadata such as: character set, description, keywords, and more.
  • The attribute charset="UTF-8" sets the character set to UTF-8 that allows the use of most characters and symbols.
  • The lang="en" attribute inside the html opening tag tells the browser the language for the webpage.

CSS

  • most common way to style a webpage is by using an external CSS style sheet
  • link file using link element in HTML's head element
  • selector defines the element or attributes to which the declarations apply
  • Declarations in CSS are totally different than declarations in HTML
    • Declarations consist of two parts:
      • CSS property
      • Property's value
    • Example of CSS rule:
      .card {
      text-align: center;
      font-size: 30px;
      }
  • The period preceding a selector designates this as a class selector. Any HTML element assigned to the class, will gain these property assignments from the CSS.
  • box-shadow is a CSS property adds shadow effects around an element's frame. You can set multiple effects separated by commas.
    • described by X and Y offsets relative to the element
    • blur and spread radius (not defined here, more later)
    • color
  • Assigning colors in CSS can be done in many ways, including by semantic references
    • can also refer to a color by its hexadecimal code
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • touch filename.filetype: to create a file in the current working directory
  • mv [file.txt] [directory_path]: to move a file or folder to any directory
    • mv file1.txt file2.txt: to rename a file since that operation is a special case of moving it
  • pwd: print working directory
  • ls: list all files and folders in the current directory
  • cd directory/path: change current working directory to specified directory/path
  • mkdir [directory_path/file]: to create new directory
  • rm [directory_path/file]: to delete specific files from directories
    • -r -R --recursive: to delete populated directories
  • git status: checks current branch
  • git log: checks current branch's history
    • it's a good idea to be in the habit of checking status and history often
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: fetch and merge commits from remote branch to local branch
  • git push: transmit local branch commits to rempte repo branch
  • git branch --move bad-branch-name corrected-branch-name: replaces your bad-branch-name with
    corrected-branch-name, but this change is only local for now.
    • git push --set-upstream origin corrected-branch-name: to push changes; let others see the corrected branch on the remote,
    • git push origin --delete bad-branch-name: to delete branch with bad name that's still present with git branch --all
    • git branch --all: to list all your branches; a * will appear next to the currently active branch
